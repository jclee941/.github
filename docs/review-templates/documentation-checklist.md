# 문서화 검토 체크리스트

> PR-Agent `jclee-bot`용 문서화 검토 전용 체크리스트
> 응답 언어: **한국어 (ko)**

---

## 1. README.md 검토

### 1.1 필수 항목
- [ ] 프로젝트/기능 목적이 명확히 설명되었는가?
- [ ] 설치 방법(Installation)이 최신 상태인가?
- [ ] 사용 예시(Usage Example)가 포함되었는가?
- [ ] 환경변수/설정 옵션이 문서화되었는가?

### 1.2 변경사항 반영
- [ ] 새로운 기능 추가 시 관련 섹션 업데이트
- [ ] API 엔드포인트 변경 시 요청/응답 예시 수정
- [ ] 의존성(dependency) 변경 시 requirements 업데이트
- [ ] Breaking change 시 Migration Guide 추가

## 2. 코드 내 문서화

### 2.1 주석 (Comments)
- [ ] 복잡한 비즈니스 로직에 "왜(Why)" 주석이 있는가?
- [ ] 핵심 알고리즘에 시간/공간 복잡도 기술이 있는가?
- [ ] 임시 조치(TODO/FIXME)에 이슈 링크가 포함되었는가?

### 2.2 독스트링 (Docstrings)
- [ ] Public 함수/클스에 Google/NumPy 스타일 독스트링이 있는가?
- [ ] 함수 파라미터의 타입과 의미가 명시되었는가?
- [ ] 반환값(return value)의 타입과 의미가 명시되었는가?
- [ ] 예외(Exception) 발생 조건이 문서화되었는가?

---

## 3. API 문서

### 3.1 엔드포인트 문서화
- [ ] 새로운 엔드포인트 추가 시 URL, Method, Auth 명시
- [ ] Request/Response 스펙이 JSON Schema 또는 예시로 제공되는가?
- [ ] 에러 응답 코드(4xx, 5xx)와 메시지가 문서화되었는가?

### 3.2 Webhook/이벤트 문서화
- [ ] 이벤트 payload 구조가 설명되었는가?
- [ ] 시그니처 검증 방법이 문서화되었는가?

---

## 4. 설정/구성 문서화

### 4.1 Configuration Files
- [ ] `.env.example`에 새로운 환경변수가 추가되었는가?
- [ ] `.toml`/`.yaml` 설정 파일의 변경사항이 문서화되었는가?
- [ ] 설정값의 기본값(default)과 유효 범위가 명시되었는가?

### 4.2 배포 문서
- [ ] Docker 관련 변경 시 `Dockerfile`/`docker-compose.yml` 업데이트
- [ ] 배포 단계(Deployment Steps)가 최신 상태인가?
- [ ] 롤백(Rollback) 절차가 문서화되었는가?

---

## 5. PR 설명 품질

### 5.1 필수 내용
- [ ] **What**: "무엇을" 변경했는가?
- [ ] **Why**: "왜" 변경했는가? (동기/문제상황)
- [ ] **How**: "어떻게" 변경했는가? (구현 방식)

### 5.2 참조
- [ ] 관련 이슈(issue) 번호가 링크되었는가?
- [ ] 스크린샷/동영상이 UI 변경 시 포함되었는가?
- [ ] 체크리스트(테스트, 문서화)가 완료되었는가?

---

## 6. 자동화 검증

> PR-Agent가 다음 항목을 자동으로 검토합니다.

- 문서화 누락 시 `docs(missing)` 라벨 제안
- README 변경 없는 API 수정 시 Warning 코멘트
- 주석 없는 Public 함수 자동 탐지
